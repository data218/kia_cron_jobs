🧮 Dashboard Calculations & Logic

MetricLogic
RenewalA VIN is classified as Renewal if it appeared in both CY and LY — the customer had a policy last year and renewed this year.
NewA VIN is classified as New if it appears in CY, has policy type "New", and was not present in LY (first-time customer).
RolloverA VIN is classified as Rollover if it appears in CY but was not in LY and does not have a "New" policy type (e.g., transferred from another insurer).
Not VisitedVINs that were present in LY (any month) but have NOT appeared in CY (any month in the selected date range).
LY #Count of VINs in the same month of the Previous Year (LY = CY - 1), categorized using the same logic (Renewal if also in LY-1, else New/Rollover).
CY #Count of VINs in the given month of the Current Year, categorized as Renewal (in LY), New (policy type "New", not in LY), or Rollover (neither).
Growth %Calculated as ((CY - LY) / LY) × 100. Colored green if positive, red if negative, yellow if flat. Shows blank for months beyond the current date.
Total RowSums up the monthly counts for each category across all visible months. Growth is computed from the aggregated totals.
Period: YTDShows all 12 months (Jan–Dec) with growth only for months up to the current date.
Period: Q1–Q4Shows only the 3 months in the selected quarter. Growth visible for months up to current date.
Period: MTDShows only the current month. Growth visible if current date is past the current month start.
Daily PerformanceShows a breakdown of New, Renewal, Rollover, and Not Visited figures for each day within the currently selected date range.
Unique VINs (Annual)Each VIN is counted only once per year, regardless of how many months it appears. This is what the info bar (LY Total / CY Total) shows — the true unique count.
Sum of Months vs UniqueMonthly columns count unique VINs per month. A VIN appearing in Jan AND Feb gets counted in both months, inflating the sum. E.g., CY sum of months = 607 vs true unique = 603 (4 VINs visited in 2 different months). This is expected — the total row reflects the correct unique count.
VIN in Multiple MonthsSame VIN + different dates + different policies across months = legitimate repeat business. Not a duplicate. Duplicates are defined as same policy number + same year.
Duplicate Policy CheckDetects rows where the same policy number appears more than once in the same year — indicating duplicate data entry.
